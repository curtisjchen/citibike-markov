# 🚲 Citibike Markov Chain Explorer

A probabilistic model of NYC Citibike station dynamics using Markov chains. Models bike flow across 2,200+ stations as a random walk, computes stationary distributions per hour and day of week, and visualizes results in an interactive dashboard.

![Dashboard screenshot](docs/screenshot.png)

## How it works

Each bike is treated as a particle doing a random walk across the station network. The transition matrix `P[i,j]` gives the probability that a bike at station `i` next appears at station `j`. The **stationary distribution** `π` gives the long-run fraction of bikes at each station — high `π_i` means bikes accumulate there, low `π_i` means bikes drain away.

Separate transition matrices are computed for each hour × day-of-week bin (168 total), with hierarchical smoothing to handle sparse bins:

```
C_smoothed(day, hour) = C_bin(day, hour) + α₁·C_day + α₂·C_global
```

A **flow score** combines the stationary distribution with the arrival/departure ratio to identify stations that need rebalancing urgently — stations that are both high-traffic and directionally imbalanced.

## Setup

**Requirements:** Python 3.11+, [uv](https://docs.astral.sh/uv/)

```bash
# clone
git clone https://github.com/yourusername/citibike-markov.git
cd citibike-markov

# install dependencies
uv sync
```

## Usage

### 1. Download data and compute outputs

Downloads Citibike trip data from S3 into `data/raw/{yyyymm}/`:

```bash
uv run src/pipeline.py --bin-minutes 15/30/60
```

This fetches monthly `.zip` files from `202001` onwards. Each file is ~100–400MB so expect the full dataset to take a while. Already-downloaded months are skipped automatically. Filters stations, builds transition matrices, computes stationary distributions, and saves outputs. You can specify time step length using `--bin-minutes` argument.

Outputs written to `outputs/`:
```
outputs/
├── pi_by_hour.npy          # (24, n_stations)
├── pi_by_day_hour.npy      # (7, 24, n_stations)
├── flow_ratio.npy          # (n_stations,)
└── stations.json           # station metadata — name, lat, lng
```

### 2. Launch the app

```bash
uv run streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## App features

- **Interactive map** — stations colored green (high π) → red (low π) by hour
- **Hour slider** — scrub through 24 hours to see how the distribution shifts
- **24-hour profile** — click any station or search by name to see its π_i across the full day
- **Flow score** — identifies stations needing rebalancing at the current hour, weighted by both traffic volume and directional imbalance
- **Station rankings** — top/bottom N stations by flow, with flow ratio and direction

## Model details

### Station filtering

Only stations in the **strongly connected component** of the trip graph are kept — this guarantees the transition matrix is irreducible and has a unique stationary distribution. ~2,213 of ~2,336 stations pass this filter.

### Stationary distribution

Computed via eigendecomposition of `Pᵀ` — the stationary distribution `π` is the eigenvector corresponding to eigenvalue 1, normalized to sum to 1.

### Hierarchical smoothing

Sparse bins (e.g. Tuesday 2:15am) are smoothed against coarser aggregates:

| Level | Weight |
|---|---|
| Hour × day bin | 1.0 |
| Day of week | α₁ = 0.1 |
| Global | α₂ = 0.01 |

This ensures every bin has a valid transition matrix even with limited data.

### Recency weighting

More recent trips are weighted more heavily using exponential decay:

```
weight(trip) = decay ^ days_ago
```

With `decay = 0.9995`, a trip from one year ago contributes ~83% as much as a trip from today. A trip from five years ago contributes ~40% as much as a trip from today.

### Flow score

```
flow_score(i, hour) = (flow_ratio(i) - 1) × π_i(hour)
```

Where `flow_ratio(i) = arrivals(i) / departures(i)`. This surfaces stations that are both high-traffic and structurally imbalanced at the current hour — the most actionable targets for rebalancing trucks.

## Limitations

- **Map click is slow** — `streamlit-folium` requires a full page rerun to return click data to Python. Use the search box for fast station selection.
- **No real-time data** — the model reflects historical patterns, not current dock counts.
- **Rebalancing contamination** — truck rebalancing trips inflate arrival counts network-wide, slightly biasing flow ratios above 1.
